# UI 设计稿说明

本文件夹是 `markdown_viewer` 的设计资料目录。
开发实现与逐元素核验只以终稿 `Markdown Viewer.dc.html` 为直接依据。
`Design System.dc.html` 用于设计师维护视觉语言、token、组件状态和扩展边界，不作为开发实现或验收依据。

## 文件清单与定位

| 文件 | 性质 | 作用 |
|---|---|---|
| `Markdown Viewer.dc.html` | **终稿(成品稿)** | app 真实那一屏的完整样子:布局 + 行为 JS + 每个元素的精确内联样式。**这是开发实现和逐元素核验的直接依据。** |
| `Design System.dc.html` | **设计系统** | 维护视觉语言、token、组件状态、扩展规则和反模式边界。**仅作为设计维护参考，不作为开发实现或验收依据。** |
| `Markdown Viewer Icon 1024.png` | 资源 | app 图标源图(1024×1024),用于生成 `AppIcon.icns`。 |
| `support.js` | 预览运行时 | 仅供 `.dc.html` 在浏览器里预览,不参与 app。 |

## ⚖️ 优先级规则(裁决铁律)

1. **开发只看终稿。** 开发实现、逐元素核验和 QA 验收只以 `Markdown Viewer.dc.html` 为依据。
2. **设计系统只服务设计维护。** 设计师可以用 `Design System.dc.html` 维护 token、组件状态、扩展规则和反模式边界，但不能要求开发以它覆盖终稿。
3. **发现冲突先做设计裁决。** 如果终稿、设计系统和核验清单出现冲突，先由设计侧决定更新哪一个文件，再让开发继续按终稿实现。
4. **例外必须落到终稿或核验清单。** 任何需要开发执行的例外，都必须体现在 `Markdown Viewer.dc.html` 或 `实现核验清单.md` 里。

> 简单说：设计系统给设计师维护方向和边界，开发实现与验收只看终稿。
